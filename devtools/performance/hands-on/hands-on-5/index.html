Understanding Network Activity in Performance Profiles

The NET section in the Performance panel shows how network activity correlates with rendering, JavaScript execution, and other browser activities. Unlike the Network panel, it helps you understand when and how network requests impact your application's overall performance.

This hands-on exercise will teach you how to analyze network-related performance issues using the Performance panel's NET feature.

Performance NET Feature Hands-On Instructions

Follow these steps to learn how to use the NET feature in the Performance panel:

  1. Open Chrome DevTools by pressing F12 or Ctrl+Shift+I (or Cmd+Option+I on Mac)
  2. Go to the Performance panel (not the Network panel)
  3. Click the gear icon ⚙️ to open the settings and ensure:
    • Network is enabled (check "Network")
    • Screenshots are enabled for better visual correlation
  4. Click the Record button ⏺️ (or press Ctrl+E)
  5. Interact with the page using the buttons below
  6. Click Stop to end the recording and analyze the results
  7. In the results, look for the Network section which shows:
    • Color-coded bars for network requests
    • How they align with rendering, JavaScript, and other browser activities
    • The impact of network requests on the critical rendering path

Pro Tips for Network Performance Analysis

      
    
        
  • Look for long-running tasks after network requests - These indicate heavy processing of
          network responses
    • 
        
  • Check for render-blocking patterns - Critical CSS/JS resources that delay first paint
    • 
        
  • Identify idle time wastage - Periods where the main thread is free but waiting for network
          requests
    • 
        
  • Use network throttling - In the Performance panel settings, simulate slower connections
        
    • 
        
  • Analyze waterfall dependencies - Resources that can't load until others complete
    • 
        
  • Correlation with frame rendering - See how network activity affects frame rate (in the FPS
          graph)
    • 
        
  • Compare multiple recordings - Try different optimization strategies and compare profiles

Performance NET Feature Analysis Guide:


      

        
1. Identifying Network Impact on Performance:

        
    
          
  • Look for blue bars in the Network section of the performance recording
    • 
          
  • Notice how they align with activity in the Main thread section above
    • 
          
  • Hover over network entries to see detailed timing information
    • 
        

      


      

        
2. Key Metrics to Watch:

        
    
          
  • TTFB (Time To First Byte): How long before the server starts responding
    • 
          
  • Resource Download Time: The blue bar's width in the NET section
    • 
          
  • Main Thread Processing: Yellow/purple bars after network requests
    • 
          
  • Request Queueing: Gray sections before download starts, indicating congestion
    • 
        

      


      

        
3. Common Performance Issues to Look For:

        
    
          
  • Render-blocking Resources: CSS/JS preventing First Paint
    • 
          
  • Long Task Bars following network requests (data processing cost)
    • 
          
  • Waterfall Dependency Chains: Resources waiting for others to complete
    • 
          
  • Connection Limits: Too many simultaneous requests to same origin
    • 
          
  • Layout Thrashing: Multiple layout recalculations after resource loads
    • 
        

      


      

        
4. Optimization Opportunities:

        
    
          
  • Preload Critical Resources: Use <link rel="preload">
    • 
          
  • Async/Defer Scripts: Prevent render blocking
    • 
          
  • Lazy Loading: Defer non-critical resources
    • 
          
  • Request Prioritization: Load important resources first
    • 
          
  • Reduce Payload Size: Compress, minify, optimize images
    •
