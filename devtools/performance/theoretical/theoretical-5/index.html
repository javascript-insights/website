Home Page

This page is designed to demonstrate the Performance tool in browser DevTools.

Network Track Analysis

Next to the Network track name, there's a legend with color-coded request types.

Render blocking requests are marked with a red triangle in the upper right corner.

Request Tooltip Information

  • Request's URL and total execution time
  • Priority or priority changes (e.g., Medium -> High)
  • Render blocking status
  • Request timing breakdown

Clicking a request draws an arrow from its initiator to the request.

Summary Tab Details

The Performance panel's Summary tab shows:

  • Initial Priority and final Priority fields
  • Detailed request timing breakdown

For more information, see Optimizing resource loading with the Fetch Priority API.

Request Timing Components

  • Left line (|–): Events up to Connection start
  • Light bar portion: Request sent and Waiting for server response
  • Dark bar portion: Content download
  • Right line (–|): Main thread waiting time

Additional timing details can be found in the Network panel's Timing tab (Right-click request and select "Reveal in Network panel").
