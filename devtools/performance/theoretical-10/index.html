Performance Tool Sections

Summary:

Overview: The Summary section provides a high-level overview of the performance recording. It includes key metrics such as total time, CPU usage, and memory usage.

Purpose: This section helps you quickly identify major performance issues and understand the overall performance characteristics of your page.

Bottom-Up:

Overview: The Bottom-Up section shows the time spent in each function, starting from the bottom of the call stack. It aggregates the time spent in each function across all calls.

Purpose: This view helps you identify which functions are consuming the most time, allowing you to focus on optimizing the most time-consuming parts of your code.

Call Tree:

Overview: The Call Tree section displays a hierarchical view of the call stack, showing how functions call each other. It breaks down the time spent in each function, including both self-time (time spent in the function itself) and total time (time spent in the function and its children).

Purpose: This view helps you understand the structure of your code execution and identify which parts of the call stack are contributing most to performance issues.

Event Log:

Overview: The Event Log section lists all events that occurred during the recording in chronological order. This includes network requests, DOM events, and JavaScript execution.

Purpose: This view helps you trace the sequence of events and understand the timing and order of operations that affect your page's performance.

Activity Types

Loading Events:

  • Parse HTML: The browser parses the HTML document.
  • Finish Loading: A network request completes.
  • Receive Data: Data for a request is received.
  • Send Request: A network request is sent.

Scripting Events:

  • Animation Frame Fired: A scheduled animation frame is executed.
  • Evaluate Script: A script is evaluated.
  • Function Call: A top-level JavaScript function is called.
  • Install Timer: A timer is created with setInterval or setTimeout.
  • Timer Fired: A timer fires that was scheduled with setInterval or setTimeout.

Rendering Events:

  • Layout: The browser calculates the layout of the page.
  • Paint: The browser paints the content to the screen.
  • Composite Layers: The browser composites layers to create the final image.

Painting Events:

  • Paint: The browser paints the content to the screen.
  • Composite Layers: The browser composites layers to create the final image.
