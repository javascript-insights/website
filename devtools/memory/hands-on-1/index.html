Memory Tool Practice

Use this website to practice tracking memory usage and identifying memory leaks using the Memory tool in browser DevTools.

Instructions for Participants

  1. Open the practice website in your browser.
  2. Open the DevTools (F12 or right-click and select "Inspect").
  3. Navigate to the "Memory" tab.
  4. Click the "Create Memory Leak" button multiple times to simulate a memory leak.
  5. Take a heap snapshot to observe the memory usage.
  6. Click the "Clear Memory" button to clear the memory.
  7. Take another heap snapshot to see the difference in memory usage.
  8. Analyze the snapshots to understand how memory is allocated and identify any potential memory leaks.
  9. Click the "Create Detached Elements" button to create detached elements.
  10. Use the "Detached elements" option in the Memory tool to identify these elements.
  11. Click the "Create Duplicated Strings" button to create duplicated strings.
  12. Use the "Duplicated strings" option in the Memory tool to identify these strings.
  13. Click the "Retain Object in Console" button to retain an object in the console.
  14. Use the "Object retained by devtools console" option to identify objects retained by the DevTools console.
  15. Use "Allocations on timeline" and "Allocation sampling" to track memory allocations over time and sample memory allocations.
  16. Use the "Comparison" option to compare different heap snapshots.
  17. Use the "Containment" option to view the containment hierarchy of objects.
  18. Use the "Object retained by detached DOM nodes" option to identify objects retained by detached DOM nodes.
