Network Metrics

  1. Requests - This indicates the total number of network requests made by the page.
  2. Transferred - This is the total amount of data transferred over the network to load the page. It includes all the bytes sent from the server to the client.
  3. Resources - This represents the total size of all resources requested by the page. The difference between "transferred" and "resources" can be due to:
    • Compression: Resources might be compressed during transfer (e.g., gzip compression), reducing the amount of data transferred.
    • Caching: Some resources might be loaded from the browser cache, reducing the amount of data transferred over the network.
  4. Finish - This is the total time taken to load all the resources and complete all network requests. It measures the time from the start of the first request to the completion of the last request.
  5. DOMContentLoaded - The DOMContentLoaded event is fired when the initial HTML document has been completely loaded and parsed, without waiting for stylesheets, images, and subframes to finish loading. This metric indicates how quickly the browser was able to parse and load the HTML content.
  6. Load - The load event is fired when the whole page, including all dependent resources such as stylesheets and images, has finished loading. This metric indicates the time taken for the entire page to be fully loaded and rendered.
